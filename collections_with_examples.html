Python Collections Overview

Тип коллекции Что это, примечания Как задать Изменяемость Пример использования Когда и почему использовать Как обращаться к элементам
Список Упорядоченная, изменяемая коллекция элементов. [] или list() Изменяемый — можно вносить правки в существующий список. tasks = ["task1", "task2"]
tasks.append("task3")		 Когда: Для упорядоченных данных, которые могут изменяться. Очередь задач, список студентов.

Почему: Быстрая вставка и доступ к элементам, но потребляет больше памяти для изменяемости.		 lst[0]
Кортеж Упорядоченная, неизменяемая коллекция. () или tuple() Неизменяемый — любое изменение создает новый кортеж. coordinates = (10, 20) Когда: Для неизменяемых данных, таких как координаты или ключи словаря.

Почему: Более эффективное использование памяти; безопасен, если не требуется изменять данные.		 tpl[0]
Множество Неупорядоченная коллекция уникальных элементов. {} или set() Изменяемый — можно добавлять или удалять элементы без создания нового множества. tags = {"python", "coding"}
tags.add("tutorial")		 Когда: Для уникальных элементов. Уникальные теги, ID пользователей.

Почему: Эффективное удаление дубликатов и поиск; меньше памяти, чем в списке.		 Нет индексации
Неизменяемое множество Аналог множества, но неизменяемое. frozenset() Неизменяемый — любые изменения создают новый объект frozenset. frozen_tags = frozenset(["python", "tutorial"]) Когда: Для фиксированных уникальных элементов. Константы, фиксированные метки.

Почему: Удобно для неизменяемых наборов уникальных данных; меньше накладных расходов.		 Нет индексации
Словарь Коллекция пар "ключ-значение". {} или dict() Изменяемый — можно добавлять и изменять пары ключ-значение без создания нового словаря. user = {"name": "John", "age": 30} Когда: Для хранения пар данных "ключ-значение". Параметры пользователя, частота слов.

Почему: Быстрый доступ по ключу, но использует больше памяти для индексации.		 dct['key']
Массив Оптимизирован для числовых данных, экономит память. array.array() Изменяемый — можно вносить изменения в существующий массив. import array
arr = array.array('i', [1, 2, 3])		 Когда: Для однотипных данных. Пример: обработка больших массивов чисел.

Почему: Меньше памяти, чем список, но только для однотипных данных.		 arr[0]
Строка Последовательность символов, поддерживает индексацию и срезы. "" или str() Неизменяемый — любое изменение создает новую строку. name = "Alice"
name.upper()		 Когда: Для работы с текстом. Имена, пароли, сообщения.

Почему: Меньше памяти, безопасна для неизменяемых текстов.		 s[0]
